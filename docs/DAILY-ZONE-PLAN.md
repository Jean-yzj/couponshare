# 當日專區規劃（含法遵配套）

版本：2026-07-16 初版；同日更新——法遵細節已落入使用條款（app/terms/page.tsx 第二、三、四、六條），上架勾選文案改為友善版。
依據：[docs/research/daily-zone-legal-research.md](research/daily-zone-legal-research.md)（台灣法規查證）＋現有資料模型（Prisma `Coupon` / `ClaimRequest` / `Transaction`）＋現有服務條款（app/terms/page.tsx）。
一句話定位：集中展示「今天就到期」的券，讓用不到的人當天送出、需要的人當天領走。純免費贈與，不涉金流。

---

## 1. 產品定義與範圍

- **當日專區 = 今天（台灣時間）到期的在架券**：`status = AVAILABLE` 且 `expiryDate` 落在台灣時間今日 23:59:59 之前。
- **只開放 GIFT（贈與），不開放 EXCHANGE（交換）**。
  理由：免費贈與幾乎立不起詐欺（送的人沒取得對價、收的人沒付出財物）；「拿假券換別人真券」是全平台唯一殘留的刑事風險情境，而當日到期的時間壓力會讓雙方來不及查證，正好放大這個情境。日後若要在當日專區開放交換，需另行評估。
- 典型供給：停車折抵券、餐飲當日券、限今日兌換的贈品券。

## 2. 實作要點（對齊現有模型，不需新 model）

- **查詢條件**：`status = AVAILABLE AND type = GIFT AND expiryDate <= 台灣時間今日 23:59:59 AND expiryDate >= now`。
- **過期判定用讀取時計算**（`expiryDate < now` 即不顯示／視同過期），不依賴 cron；若專案已有過期排程則沿用。
- **上架流程新增三件事**：
  1. 強制勾選聲明（文案見 §4-2，不勾不能發布）。
  2. 類別黑名單攔截（§5）。
  3. 停車券提示（§4-3，分類為停車或標題含「停車」時顯示）。
- **檢舉入口**：券詳情頁一鍵檢舉 → 進 admin 待處理。若尚無檢舉機制，這是本功能必做項——它是平台「善良管理、無明知」的最重要證據。
- **下架動作**：沿用條款第六條的內容移除權。若現有 status 流轉（DRAFT → AVAILABLE → CLAIMED → EXPIRED）沒有「管理員移除」的對應狀態，需補一個 `REMOVED`（或等效標記），不要用 EXPIRED 冒充，留存紀錄要分得清楚。
- **領取流程沿用** ClaimRequest → 擁有者批准 → 完成後揭露 barcode，不另做即時搶券。

## 3. 品牌方下架 SOP

**入口**：條款/隱私頁的聯絡信箱（iamlazybear2023@gmail.com）；頁尾 IG 私訊視同來函。

**原則：不爭執、先配合。** 來函即下架是成本最低的回應，也是最好的護身符。平台不是券契約的當事人，配合下架後對方幾乎沒有繼續追究的空間。

**流程（收到來函起 24 小時內完成）**：
1. 確認來函者是以公司/品牌名義聯絡、有可回覆的聯絡方式即可。不要求嚴格舉證——寧可先下架。
2. 下架該品牌所有在架券，記錄下架清單（券 ID、數量）。
3. 依範本回覆（§3-1）。
4. 視情況將該品牌加入上架管制清單（brand blocklist），之後上架選不到／關鍵字攔截。
5. 留存紀錄：日期、品牌、來函全文、處理動作、下架筆數。email 原文保留不刪。

**若收到的是存證信函（紙本）**：處理方式相同（下架＋回覆），原件保存。只有在對方進一步要求金錢賠償時，才需要諮詢律師，在那之前不用。

## 4. 揭露與聲明文案（可直接貼進 UI）

### 4-1. 專區頁首固定揭露

> 當日專區的券由使用者即時分享，**當天到期、先領先用**。平台不查驗券的有效性，無法保證每張券都能成功使用，請自行評估後領取。

### 4-2. 上架強制勾選聲明（checkbox，不勾不能發布）

> 這張券是我自己取得、還沒用過的。如果有使用限制（像是限本人、綁定車牌或發票），我已經在說明裡寫清楚，讓領到的人可以放心使用。

勾選框下方小字（含連結）：

> 發布即表示同意〈使用條款〉的分享者聲明。

（設計說明：法律重話——自負全責、違反發行條款的責任歸屬——已全部寫進使用條款第二條（2.2 聲明保證、2.3 責任歸屬、2.4 勾選即確認），UI 文案只保留友善的分享語氣＋條款連結，法律效果相同。另外不要求使用者聲明「券未標示禁止轉讓」——那會把多數停車券擋掉、也逼人做不實聲明；改為要求「如實揭露限制」（條款 2.2(c)），把責任清楚留在上架者身上，同時保住供給。）

### 4-3. 停車券提示（上架時顯示）

> 提醒：綁定車牌或發票的停車折抵券，換人後通常無法使用。請確認這張券**持券即可折抵、不限車牌**，並在說明中註明使用方式。

### 4-4. 領取前提示（claim 對話框）

> 這張券**今天到期**，領取後請儘速使用。平台不保證券可成功兌用，若發現異常請使用檢舉功能。

## 5. 類別黑名單

**硬性封鎖（網售/交易本身違法，分類選單不提供＋關鍵字攔截）**：
- 菸品（含電子煙、加熱菸）——菸害防制法，網路販售全面違法
- 酒類——菸酒管理法禁止網路販售
- 藥品、醫療器材——藥事法，非藥商網售違法
- 彩券、博弈——公益彩券發行條例禁止受讓

**預設排除（高糾紛或監理灰區，先不開放）**：
- 醫美療程券
- 金融商品、投資相關
- 成人商品

**實作**：分類選單直接不提供上述類別；標題/描述關鍵字命中時軟性攔截（顯示原因、可申訴），避免誤殺。

**條款依據**：黑名單已寫入使用條款第三條（3.2 禁止刊登項目、3.1 無償原則）；品牌下架機制寫入第六條 6.3（對外承諾「合理期間內儘速處理」，內部 SOP 仍以 24 小時為準）；記名/顯然無法轉用的票券可依 6.2(d) 移除。

## 6. 行銷紅線（一句話）

宣傳語氣守住「用不到的券不浪費、分享給需要的人」的互助定調；**禁止任何「教你繞過限本人／百貨限制」的表述**——法律上百貨很難動平台，但這種語氣會把品牌法務主動引出來。
