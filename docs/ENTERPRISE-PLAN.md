# CouponShare 企業合作企劃（內部版）

版本 2026-07-07。用途：對外銷售的依據＋產品開發的路線圖。對外只給「報價信範本」與 pitch 網站，本文件不外流。

---

## 一、定位與核心論述

CouponShare 的企業合作不是賣廣告版位，是賣「**官方福利券投放**」。

一句話論述：**使用者進來就是為了找券、領券、換券——品牌優惠出現在這裡，不是被滑過的廣告，而是使用者主動想申請的福利。**

對企業的三個價值：

1. **場景對了**：優惠出現在使用者有領券意圖的時刻（瀏覽、搜尋、管理錢包、券過期時），不是打斷式曝光。
2. **數據是真的**：曝光、點擊、申請、領取每一層都可追蹤。企業買到的不是「觸及」，是「有多少人真的想要這個優惠」。
3. **族群精準**：以學生與年輕族群為主的活躍社群（16,000+ 使用者），適合校園店家、飲料咖啡甜點、課程與 App 導流。

適合對象：飲料/咖啡/甜點品牌、超商零售、外送平台、線上課程、校園活動、職涯服務、App 註冊活動、電商優惠碼。

## 二、關鍵架構決策：企業券 ≠ 一般券

這是整個方案最重要的一條，直接影響後台、報價與數據：

| | 一般使用者券 | 企業官方福利券 |
|---|---|---|
| 來源 | 個人上傳 | 合作企業上架 |
| 發放邏輯 | **一張券送一個人** | **一個活動券可被多人申請**（設名額） |
| 目的 | 互助、減少浪費 | 品牌曝光、導流、收名單 |
| 數據 | 無 | 曝光/點擊/申請/領取全程追蹤 |
| 商業化 | 永不販售 | 依方案收費 |

企業券獨立一套資料模型與邏輯，**不要**塞進現有的贈送/交換流程。

## 三、對外方案與定價（最終版）

### 3.1 四個方案

**早鳥試用｜NT$5,000 / 月**（限前 10 家）
- 每月 5 張官方福利券，每張最多 100 次申請，月總申請上限 500
- 曝光：一般 Feed、搜尋結果、錢包底部推薦
- 基礎成效數據；合作期間可調整券內容 2 次
- 條件：需提供回饋、同意作為對外案例

**標準導流｜NT$8,000 / 月**（主力銷售）
- 每月 10 張，每張最多 200 次申請，月總上限 2,000
- 支援直接領取、留言申請；可看申請者留言
- 曝光：Feed 中段、熱門票券區、搜尋優先、錢包推薦、詳情頁推薦
- 每月成效摘要；可調整券內容 5 次

**精選品牌｜NT$18,000 / 月**
- 每月 25 張，每張最多 500 次申請，月總上限 12,500
- 加上：品牌審核模式、申請問題設定、品牌專屬卡片、指定分類置頂、通知中心低頻提醒
- 完整成效報告＋文案優化建議；可調整 10 次

**品牌專案｜NT$30,000 起 / 檔**（2–4 週）
- 專屬活動頁、券數與名額客製、任務解鎖/抽選/問卷/排行榜
- 可搭配社群曝光（Threads / Dcard / LINE 社群）
- 完整活動成效報告＋活動企劃建議

### 3.2 加購項目

| 項目 | 價格 |
|---|---|
| 首頁大型活動曝光 | NT$5,000–15,000 / 週 |
| LINE 社群曝光 | NT$2,000–8,000 / 則 |
| Dcard / Threads 內容曝光 | 依帳號另計 |
| 活動頁製作 | NT$8,000–30,000 / 頁 |
| 客製任務機制 | NT$10,000 起 / 次 |

### 3.3 定價策略

- 初期**只主推早鳥（5,000）與標準（8,000）**，精選與專案對外一律「客製報價」——先累積案例，別讓高價方案嚇跑第一批客戶。
- pitch 網站只露出早鳥與標準的價格；精選/專案寫「聯繫取得報價」。
- 有 3–5 個成功案例後，精選方案再往 NT$18,000–30,000 / 月調。

## 四、申請模式（賣點之一：不只是曝光，是互動）

1. **直接領取**：點了就拿。適合折扣碼、導購連結。轉換最快。
2. **留言申請**：留一句話才能申請（「你為什麼想領？」）。適合限量體驗券，順便收使用者洞察。
3. **品牌審核**：企業挑人發放。適合高價值試用、品牌大使、抽選。
4. **任務解鎖**：完成任務才能領（問卷、追蹤社群、加 LINE、分享、邀請朋友）。只放在品牌專案方案，不進低價月費。

## 五、曝光原則（守住平台體驗）

**鐵律：一般券 70% / 企業券 30%**，首頁 Feed 每 5 張一般券最多插 1 張企業券，或集中在「官方福利」區塊。

曝光位置清單：首頁福利區塊、一般列表上方置頂卡、熱門票券區、分類/搜尋優先、錢包底部推薦、票券詳情頁「你可能也需要」、通知中心低頻提醒（精選以上）。

標籤用語：用「官方福利」「品牌合作」「限量申請」「校園限定」；**不用**「廣告」「業配」「Sponsored」。必要揭露時在卡片下方小字「品牌合作福利券」。

呈現鐵律：企業券長得像一張券（與一般券同風格＋合作標籤），**不是** banner；不做彈窗、不擋內容、不每頁轟炸。

## 六、成效數據與月報

每月給企業一份報告，包含：

1. 總覽：上架券數、總曝光、總點擊、總申請、總領取、平均申請率/領取率、剩餘名額
2. 單張券成效表（曝光/點擊/申請/領取/申請率/剩餘）
3. 曝光位置成效比較（Feed vs 錢包 vs 詳情頁 vs 搜尋）
4. 留言申請的使用者洞察摘要（最常見申請理由、關鍵字、優惠吸引點）

## 七、銷售工具包

### 7.1 一句話開場（對誰都適用）

「CouponShare 的使用者進來就是為了找優惠券，所以品牌優惠不會像廣告被滑過——它是使用者主動申請的福利，而且每一層轉換都有數據。」

### 7.2 分對象話術

- **校園店家**：「比發傳單準——這裡的學生本來就是為了找優惠進來的，你的券會出現在他們搜尋和瀏覽的地方，申請了才算數。」
- **一般品牌**：「不是買曝光，是把優惠放進一個使用者本來就有領券需求的場景，用申請數和領取數看成效。」
- **大型品牌**：「低成本測試不同優惠內容對年輕族群的吸引力——哪種券被搶著申請、哪種沒人要，一個月的數據就看得出來。」

### 7.3 開發訊息範本（初次接觸）

> 您好，我們是 CouponShare，一個讓使用者分享、贈送與申請兌換券的平台。
>
> 我們正在開放第一批品牌合作，讓企業以「官方福利券」的形式，把優惠自然放進使用者正在找優惠的場景中。不同於一般廣告版位，這裡的使用者本來就有領券需求，因此品牌優惠能被主動申請，也能追蹤實際申請與領取成效。
>
> 如果貴品牌近期有優惠券、折扣碼、新品體驗或校園推廣計畫，我們有一個限量的早鳥合作方案，很適合先小規模測試成效。方便的話，我把完整方案與報價寄給您參考。

## 八、合約要點（條款草稿）

1. **合作內容**：依所選方案取得上架張數、申請名額、曝光位置與數據報告。
2. **素材保證**：企業保證素材、商標、優惠內容有合法使用權。
3. **兌換責任**：優惠無法兌換、資訊錯誤、提前終止之爭議由企業負責處理。
4. **平台裁量**：平台可依使用者體驗調整呈現位置與樣式；除專案另有約定，不保證特定曝光/點擊/申請數。
5. **數據性質**：報告數據僅供成效參考，不構成保證。
6. **下架權**：優惠不實、大量無法使用回報、違法內容、誤導文案、未授權素材、傷害使用者體驗——平台可暫停或下架。
7. **案例授權**：經雙方同意，合作成果可作為平台對外案例（早鳥方案為必要條件）。

## 九、產品實作路線（誠實現況）

### 現況（2026-07-07）

- ✅ **企業合作窗口已上線**：`/business`（隱藏頁、noindex，只從 pitch 網站或私訊連過去）收 姓名/Email/電話/LINE ID → 存 DB → 站內通知管理員 → 後台「企業合作」分頁管理名單（待寄報價/已寄報價）。
- ❌ **企業券系統尚未開發**——多人申請、名額、曝光加權、數據追蹤都還沒有。現有 Coupon 模型是一對一贈送，不可混用。

### 第一階段：最快可賣版本（成交第一家再動工）

1. `BrandCoupon` 資料模型（多人申請、名額、期限、CTA 連結）
2. Admin 手動代上架（不做企業後台）
3. 首頁官方福利區塊＋錢包底部推薦兩個曝光位
4. 使用者申請流程（直接領取＋留言申請兩種模式）
5. 曝光/點擊/申請計數
6. Admin 匯出簡易數據 → 手動整理成月報

原則：**平台代上架＋人工月報**，先驗證企業願付費、使用者會申請，再投資自動化。

### 第二階段：企業自助後台

企業登入、自行上架/編輯券、看申請名單、看數據儀表板、匯出報告。

### 第三階段：專案活動工具

任務解鎖、抽選、問卷、排行榜、品牌活動頁、自動報表、社群導流追蹤。

### 資料模型備忘（開發時參考）

`brands`、`brand_plans`（方案與額度）、`brand_coupons`（含 application_mode: DIRECT_CLAIM / MESSAGE_APPLICATION / BRAND_REVIEW / TASK_UNLOCK）、`brand_coupon_applications`（status: PENDING/APPROVED/REJECTED/CLAIMED/EXPIRED）、`brand_coupon_metrics`（按日按位置計數）。權限：企業只能管自己品牌；使用者個資非同意不給企業；名單匯出需標示用途。
